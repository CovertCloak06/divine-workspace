## Executive Summary

Divine Toolkit is a unified Android application with **Chat as the primary interface** and a **sidebar menu** that provides access to two additional "sub-app" sections:

| Section | Purpose | Access |
|---------|---------|--------|
| **PKN Chat** | AI conversation assistant (MAIN UI) | Default view |
| **AI IDE** | Agent-powered coding environment | Sidebar â†’ IDE |
| **OSINT Suite** | Intelligence gathering tools | Sidebar â†’ OSINT |

Each section operates like its own app but is accessed from the sidebar menu. All sections share the same backend (Ollama, agents, memory).

**Design Reference:** The OSINT section is modeled after the existing **DVN Toolkit** app (`/home/gh0st/dvn/scripts/android_app/`) which has 130+ tools in a card-based UI.

## DVN Toolkit Reference

The OSINT section should be modeled after the existing **DVN Toolkit** app:

**Location:** `/home/gh0st/dvn/scripts/android_app/`

**Key Design Elements to Replicate:**

| Feature | DVN Toolkit Implementation | Apply to OSINT Section |
|---------|---------------------------|------------------------|
| **Card Grid** | 2-column tool cards with icons | Same layout for OSINT tools |
| **Tool Detail** | Rich docs + dynamic forms | Same for each OSINT tool |
| **Output Screen** | Terminal-style with progress | Same for recon results |
| **Themes** | 5 themes (Cyberpunk default) | Share with main app |
| **Favorites** | Horizontal scroll section | Quick access to common tools |
| **Recents** | Last 5 used tools | Last investigations |

**DVN Toolkit Screens to Reference:**
- `app/screens/dashboard.py` - Card grid layout
- `app/screens/tool_detail.py` - Documentation + forms
- `app/screens/output.py` - Results display
- `app/components/tool_card.py` - Card widgets
- `app/components/form_builder.py` - Dynamic form generation

**Note:** DVN Toolkit stays as a separate standalone app. We're using its design patterns for the OSINT section within Divine Toolkit.

### Key Features

#### 1. File Explorer (Left Panel)
- Tree view of project files
- Create/rename/delete files
- Search files by name
- Git status indicators (modified, new, deleted)

#### 2. Code Editor (Main Area)
- Syntax highlighting (50+ languages)
- Line numbers
- Tab support (multiple open files)
- Find/replace with regex
- Undo/redo
- Auto-indent

#### 3. AI Assistant Panel (Floating)
- Inline suggestions as you type
- Triggered by comments: `# AI: write a function to...`
- Code explanations on selection
- Refactoring suggestions
- Error detection and fixes

#### 4. Terminal (Bottom Panel)
- Full bash terminal
- Command history
- Output capture for AI analysis
- Quick commands: run, test, lint

### AI IDE Actions

| Action | Trigger | Agent Used |
|--------|---------|------------|
| Code completion | Typing + pause | Coder |
| Explain selection | Select + "Explain" | General |
| Refactor | Select + "Refactor" | Coder |
| Fix error | Click error marker | Coder |
| Generate from comment | `# AI: ...` comment | Coder |
| Write tests | "Generate tests" button | Coder |
| Optimize | "Optimize" button | Reasoner + Coder |

| Module | Tools | Purpose |

## Technical Requirements

### Android
- Minimum SDK: 24 (Android 7.0)
- Target SDK: 34 (Android 14)
- Permissions:
  - INTERNET (API calls)
  - READ_EXTERNAL_STORAGE (file access)
  - WRITE_EXTERNAL_STORAGE (file save)
  - CAMERA (image capture for OSINT)

### Termux Integration
- Must work alongside Termux app
- Access to Termux filesystem (`~/`)
- Shared storage with Termux
- Can launch Termux commands

### Backend
- Python 3.11+ (Flask)
- Ollama daemon running
- Port 8010 (Flask), 11434 (Ollama)

### UI Framework
- React Native or Flutter (cross-platform)
- Dark theme (cyberpunk aesthetic)
- Accent color: Cyan (#00FFFF)
- Background: Dark (#0a0a0f)

### Typography
- Headers: "Orbitron" or similar cyberpunk font
- Body: System font (Roboto on Android)
- Code: "Fira Code" or "JetBrains Mono"

### Touch Targets
- Minimum: 44x44 pixels
- Comfortable: 48x48 pixels
- Spacing: 8px minimum between targets

### Animations
- Transitions: 200ms ease-out
- Loading: Pulsing cyan glow
- Success: Brief green flash
- Error: Brief red shake
